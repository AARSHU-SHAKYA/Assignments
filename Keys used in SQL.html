KEYS

  • KEYS USED IN DBMS
  • What is a key?

    • Keys are one of the basic requirements of the relational database model. It is widely used to identify the rows uniquely in the table.

  • Candidate key

    • The minimal set of attributes that can uniquely identify a tuple is known as a candidate key. For example: serial number, ID number, etc. It is one of the keys with no repeated data. It is a minimal set of attributes that can uniquely identify a record. It can contain unique values. A table can contain multiple candidate keys.

  • Primary key

    • There can only exist one primary key in a table. There can be more than one candidate key in relation out of which one can be chosen as the primary key. It is a unique key. It can identify only one record at a time. It cannot be null.

  • Super key

    • The set of attributes that can uniquely identify a tuple is known as super key. A super key is a group of single or multiple keys that identifies rows in a column. It supports null value.

  • Alternate key

    • The candidate key other than the primary key is called an alternate key. IN other words, all the keys which are not primary keys are called alternate keys. It is a secondary key. It contains two or more fields to identify two or more records. Those values are repeated.

  • Foreign key

    • If an attribute can only take the values which are present as values of some other attribute, it will be a foreign key to the attribute to which it refers. The relation which is being referenced is called referenced relation and the corresponding attribute is called referenced attribute. It acts as a primary key in one table and it acts as secondary key in another table.

  • Composite key

    • Sometimes, a table might not have a single column attribute that uniquely identifies all the records of a table. To uniquely identify rows of a table, a combination of two or more columns or attributes can be used. It acts as a primary key if there is no primary key in the table. Two or more attributes are used together to make a composite key.
