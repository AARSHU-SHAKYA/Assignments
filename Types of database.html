TYPES OF DATABASE

  • DATABASE DESIGN
  • Database Schema Diagram

    • A database Schema is considered the "blueprint" of a database which describes how the data may relate to other tables or other data models. However, the schema does not actually contain data. A sample of data from one database at a single moment in time is known as startup is instance.

  • Entity-relationship (ER) diagram

    • An entity relationship diagram is a diagram That represents relationships among entitles in a database. It is commonly known as an ER diagram. An ER diagram in DBMS plays a crucial role in designing the database. Today's business world previews all the requirements demanded by the users in the form of an ER diagram.

  • TYPES OF DATABASE
  • Hierarchial Database

    • It is a type of database that uses a hierarchal model to organize data. In a hierarchal database, data is organized into a tree like structure, with each records represented as a node in a tree. Each node is connected to one or more child nodes, and child node can have its own set of child nodes, creating hierarchy of nodes.

  • Network Database Model

    • A network database uses a network model to organize data. In a network database, data is organized into a series of interconnected records, with each record representing an entity and relationships between the entities represented as lines connecting the records.

  • Relational Database

    • These are the most widely used type of databases, and they store data in tables that are related to each other through common trees or indexes.

  • Object-oreinted Database

    • In an object-oriented database, data is organized into objects, which are self-contained entities that contain both data and the methods that operate on the data. Object oriented databases are designed to support the creation and management of complex data structures, and they are often used in applications that require the manipulation of large amount of structured and semi structured data.
